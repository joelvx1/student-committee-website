Optimisation

Using large, high-quality images noticeably slowed down the loading time of my website. This effect would be worse on devices with slower internet connections. To mitigate this I used images with a smaller file size (about 400kb). These still looked good and the loading time is imperceptable.

Debugging

I used the following HTML and CSS validators to check my code:

  • https://validator.w3.org
  • https://jigsaw.w3.org/css-validator

These helped me make my code more efficient by removing unnecessary white space which can sometimes cause formatting issues. They also suggested I replace ID's that were used multiple time with classes instead. Using ID's multiple times does not cause errors in the browser as they are lenient and can workaround this, but it is bad practise.

Browser Testing

I tested my website on three different browsers using their built in emulation tools: Google Chrome (v119), Microsoft Edge (v119), Firefox (v119). There was no difference in the formatting between all three browsers. The only difference I noticed was that Firefox displayed my website more saturated (which I tested with a colour picker) which is due to a current bug in this Firefox version. However this has absolutely no impact on using the website.

Accessibility

After running the Wave Report tool on my website, the only recommended action I had to take was to incorporate lang="en" in the html tag, so screenreaders and other accessibility tools can recognise the language of my webpage. I have tested that my website can be easily navigated using only the keyboard (tab key) for those who cannot use a mouse, and a visible keyboard focus indicator is automatically applied by browsers for accessibility. All the elements on the 'Contact' form have labels so they can be understood by screenreaders.
